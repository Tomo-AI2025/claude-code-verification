# Claude Code の死角 — AI コード生成で誰も教えてくれないこと

> **Claude Code は指示の平均 ~60% しか実装しません。** 残り40%は「サイレント・フェイラー」— 関数は定義されるが呼ばれない、数値パラメータが勝手に変わる、コミットメッセージに「実装完了」と書かれるが実際は未完成。

本リポジトリは、実運用コードベース（自動株式売買システム）での **16回の実証検証ラウンド** を通じて発見した **6つの体系的失敗パターン** と **5つの黄金法則** を記録しています。

## なぜこれが重要か

Claude Code、Cursor、Copilot、その他あらゆるLLMコーディングツールを使う全開発者が、同じ「見えない問題」に直面しています。**コードは完成したように見えるが、重要な配線が欠落している**のです。

これを **「サイレント・フェイラー（沈黙の失敗）」** と呼びます。ファイル内に関数が存在し、docstringも適切で、ユニットテストもある。でも他のモジュールがその関数を一度も呼ばない。システムが正常に動いていると思い込んだまま、実は何も機能していない。

**これは Claude Code のバグではありません。** LLMがコードを生成する際の構造的な限界であり、全てのAIコーディングツールに共通する現象です。

## 6つの失敗パターン（頻度データ付き）

| # | パターン | 頻度 | 危険度 | たとえ話 |
|---|---|---|---|---|
| 1 | **サイレント・フェイラー** — 関数を定義するが他から呼ばない | 69%（16回中11回） | ★★★★★ | インターホンを壁に付けたが線がつながっていない |
| 2 | **数値改変** — パラメータを勝手に変更（$1,000→$10,000） | 25%（16回中4回） | ★★★★☆ | 「砂糖大さじ1」が「大さじ10」になる |
| 3 | **末尾無視** — 指示の後半を読み飛ばす | 19%（16回中3回） | ★★★☆☆ | お使いメモの下3行を見ない |
| 4 | **前提ファイル欠落** — コードが参照するファイルを作らない（RSA鍵） | 13%（16回中2回） | ★★★★★ | 鍵穴を作ったが鍵を作り忘れた |
| 5 | **設定のみ** — 設定辞書を定義するが実行コードがない | 50%（16回中8回） | ★★★☆☆ | メニューは豪華だが料理人がいないレストラン |
| 6 | **コミットの嘘** — 「統合済み」とコミットするが実際は未統合 | 6%（16回中1回） | ★★★★☆ | 日記に「宿題やった」と書いたがやっていない |

## 5つの黄金法則

### 法則1：「定義」と「配線」は別コマンドで指示せよ
```
# ダメ — 配線が省略される
「cost_manager.pyにcalculate_net_profit()を追加してください」

# 良い — 2段階で指示
コマンド1:「cost_manager.pyにcalculate_net_profit()を追加してください」
コマンド2:「executor.pyの売却約定後にcost_manager.calculate_net_profit()を呼び出してください」
```

### 法則2：呼び出し場所はファイル名・関数名・タイミングまで明示せよ
```
# ダメ — 曖昧すぎる
「scoring.pyに統合してください」

# 良い — 具体的な場所を指定
「src/analysis/scoring.pyのscore_stock()関数の120行目に
score += whale_tracker.calculate_score() を追加してください」
```

### 法則3：必ず「ペア検証」を行え（定義のgrepと呼出のgrep）
```bash
# 定義の存在を確認
grep -n "def evaluate" src/trading/kill_switch.py
# → 152: def evaluate(self, current_capital)  ✅

# 呼出の存在を確認
grep -rn "evaluate" src/ --include="*.py" | grep -v "def evaluate"
# → executor.py:277: self._kill_switch.evaluate(current_capital)  ✅

# 呼出が0件 → サイレント・フェイラー — 関数は存在するが何も起きない
```

3. 世界初の**「ペア検証」方法論**（定義grep＋呼出grep）
4. 世界初の**「仕様先行更新」ルール**（CLAUDE.md更新→コード変更の順序強制）
